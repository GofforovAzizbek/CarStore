1. STACK (Call Stack)
Stack — bu xotiraning tez ishlaydigan qismi.
Unda primitive turlar va funksiyalar chaqiruvi saqlanadi.

Primitive turlar:
- Number
- String
- Boolean
- null
- undefined
- Symbol
- BigInt

Misol:
let a = 10;
let b = a;
b = 20;

Natija:
a = 10
b = 20

Sabab:
a ning qiymati nusxa bo‘lib stack’da saqlanadi.

Function Stack misoli:
function one() {
  two();
}

function two() {
  console.log("Hello");
}

one();

Call Stack ketma-ketligi:
one()
two()
console.log()

2. HEAP
Heap — murakkab obyektlar saqlanadigan xotira.
Sekinroq, lekin katta hajmli.

Reference turlar:
- Object
- Array
- Function

Misol:
let obj1 = { name: "Ali" };
let obj2 = obj1;

obj2.name = "Vali";

Natija:
obj1.name = "Vali"
obj2.name = "Vali"

Sabab:
Heap’da faqat manzil (reference) saqlanadi.

3. STACK + HEAP BIRGA ISHLASHI
let user = {
  name: "Aziz",
  age: 20
};

Stack:
user -> heap’dagi obyekt manzili

Heap:
{ name: "Aziz", age: 20 }

4. TAQQOSLASH JADVALI

STACK:
- Tez ### kak bi ssd dip tushundim
- Kichik xotira
- Primitive qiymatlar
- Function call

HEAP:
- Sekinroq ### hhd
- Katta xotira
- Object, Array
- Reference asosida ishlaydi

5. MUHIM ESLATMA
Primitive → copy by value
Object / Array → copy by reference

6. REAL MISOL
function change(x, obj) {
  x = 5;
  obj.age = 30;
}

let num = 10;
let person = { age: 20 };

change(num, person);

Natija:
num = 10
person.age = 30

Sabab:
num → stack (copy)
person → heap (reference)

Xulosa:
Stack — tez va vaqtinchalik
Heap — katta va reference asosida

Yodlab olish uchun:
Primitive = Stack
Object = Heap
